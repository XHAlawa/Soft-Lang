# Soft Runtime Philosophy v2

This document is the architectural contract of the Soft language.

## Vision

- Soft is a programming language, not a framework.
- The compiler owns infrastructure; the developer owns business logic.
- Every feature must reduce business boilerplate.

## Core Rules

- The developer writes intent.
- The compiler writes implementation.
- The runtime executes metadata only.
- No runtime template parsing.
- No runtime dependency discovery.
- No Virtual DOM.
- No dirty checking.
- No signals API exposed to developers.

## Compiler Ownership

- Discover pages from @Page.
- Generate ApplicationManifest.
- Generate bootstrap.
- Generate routing.
- Generate DI registrations.
- Generate dependency graph.
- Generate runtime metadata.
- Emit TypeScript only.

## Runtime Ownership

- Instantiate pages.
- Instantiate components.
- Wrap state objects with generated proxies.
- Dispatch lifecycle.
- Dispatch events.
- Execute dependency graph.
- Patch DOM directly.

## Reactive Engine

- Every state object becomes a proxy automatically.
- Developers never call reactive(), signal(), ref(), observable().
- Setter interception is the only source of UI updates.
- Arrays and collections are intercepted.
- onChange is generated by the compiler.

## Rendering

- Dependency -> DOM Node mapping is generated at compile time.
- Updating customer.name updates only dependent DOM nodes.
- No component rerender.
- No page rerender.
- No tree diff.

## Bootstrap

- `soft dev` discovers all .s files.
- Compiler generates AppManifest.
- Compiler generates bootstrap.
- Compiler invokes tsc.
- Browser runs generated output.

## Architecture Contracts

- Business objects are state.
- State is strongly typed.
- Generated code is hidden.
- Generated code is deterministic.
- Runtime remains under 30KB target.

## Anti Patterns

- Manual render().
- new Page().
- Manual routing.
- Manual DI registration.
- Manual change notification.
- Framework-specific stores.

## Future

- SSR via new generator.
- WASM via new generator.
- Native via new generator.
- Soft IR remains unchanged.

## Mandatory Rules
1. The compiler owns infrastructure rule #1. If a responsibility can move from runtime to compile time, prefer compile time.

## Runtime Pipeline
- Source (.s)
- Lexer
- Green Tree
- Red Tree
- Semantic Model
- Binder
- Dependency Analyzer
- Soft IR
- TypeScript Emitter
- TypeScript Compiler
- Generated JavaScript
- Runtime Bootstrap
- Proxy Engine
- Dependency Dispatcher
- Rendering Engine
- Browser

## Final Principle
The developer should feel like writing plain TypeScript business code. Every framework concern belongs to the compiler.
