# Soft Reactive Engine & Runtime Internals
Version: 1.0

## Purpose

This document explains how the Runtime executes a Soft application.

It intentionally focuses on implementation details instead of philosophy.

---

# 1. Runtime Modules

The runtime is composed of small independent modules.

- ApplicationHost
- Router
- ComponentManager
- ProxyFactory
- ChangeDispatcher
- DependencyExecutor
- RenderEngine
- DOMRegistry
- EventDispatcher
- ServiceContainer
- ErrorPipeline
- FormRuntime

Each module has one responsibility.

---

# 2. Compiler Output

The compiler generates metadata.

Example

Page Metadata
Component Metadata
Dependency Graph
Render Instructions
Route Table
Application Manifest

The runtime never builds those.

It only consumes them.

---

# 3. Boot Process

soft dev

↓

Compiler

↓

generated/App.ts

↓

Browser

↓

ApplicationHost.start()

ApplicationHost performs

1. Load Manifest
2. Create Router
3. Register Services
4. Register Components
5. Navigate to current URL
6. Create Page Instance
7. Wrap State Objects
8. Mount DOM
9. Start Event Loop

Developer never performs these steps manually.

# 5. State

Every class instance is state.

Example

class Customer{

    name=""
    age=0

}

Compiler automatically wraps every reachable state object.

The developer never calls any runtime API.

---

# 6. ProxyFactory

ProxyFactory is one of the most important runtime modules.

Input

Customer Instance

Output

Proxy<Customer>

The Proxy intercepts

- property assignment
- delete
- array mutations
- nested objects

Example

customer.name = "Ahmed"

The assignment never reaches the original object directly.

The proxy intercepts it first.

---

# 7. Generated Proxy

The compiler generates code similar to:

const state = ProxyFactory.create(instance, metadata);

The developer never sees this code.

The proxy already knows the Dependency Graph Id generated by the compiler.

No runtime reflection is required.

---

# 8. Property Write Pipeline

customer.name = "Ali"

↓

Proxy.set()

↓

ChangeDispatcher.publish()

↓

Execute property subscribers

↓

Execute object subscribers

↓

DependencyExecutor.execute()

↓

RenderEngine.patch()

↓

DOM updated

Every step is synchronous unless explicitly async.

---

# 9. onChange

The compiler generates strongly typed APIs.

customer.onChange(handler,p=>p.name,p=>p.age)

customer.name.onChange(handler)

The runtime only executes them.

The runtime never discovers properties dynamically.

Subscriber Context

stopHere()

Stops remaining subscribers.

stopUIRender()

Skips renderer execution.

continue()

Allows normal execution.

---

# 10. Arrays

Arrays are wrapped automatically.

Intercept

push
pop
splice
shift
unshift
sort
reverse

Each operation produces a Change Event.

Compiler metadata determines affected template nodes.

No array diff algorithm exists.

---

# 11. Nested Objects

customer.address.city="Cairo"

Every nested object is wrapped lazily.

The runtime creates nested proxies only when required.

---

During initial render every generated element receives an internal id.

# 15. Events

Compiler generates event registration.

Example

@click:save(customer)

Generated

button.onclick = () => page.save(customer);

Runtime dispatches.

Nothing is parsed.

# 17. Forms

Compiler generates Form Metadata.

Runtime tracks

Touched

Dirty

Valid

Invalid

Validation Messages

Binding Metadata

The runtime does not inspect HTML.

---

# 18. Runtime Rules

The runtime MUST NOT

- Parse HTML
- Parse TypeScript
- Parse Expressions
- Build Dependency Graphs
- Register Pages
- Register Routes
- Discover Components
- Perform Reflection Scans

All information comes from compiler metadata.

---

# 19. Performance

DOM references are cached.

Proxy instances are reused.

Batch DOM updates.

Never rerender complete components.

Never rerender complete pages.

Only execute compiler-generated operations.

---

# 20. Mental Model

Think of the runtime as a tiny virtual machine.

Compiler:

Produces executable metadata.

Runtime:

Executes executable metadata.

Business objects never communicate directly with the DOM.

They communicate through:

Proxy
→ ChangeDispatcher
→ DependencyExecutor
→ RenderEngine

That pipeline is the heart of Soft.
